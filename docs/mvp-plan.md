# 网页对照翻译浏览器插件 MVP 计划

## 1. 概述

本项目目标是实现一个浏览器插件，用于对用户选中的网页内容容器进行原文和目标语言的对照翻译。

MVP 使用 WXT、React 和 TypeScript 开发。不使用浏览器 Side Panel，不实现翻译缓存。每次用户在 popup 中点击“开始翻译”，插件都会重新抽取当前仍然有效的已选容器内容，并重新调用配置好的 LLM 进行翻译。

产品 UI 分为两部分：

- 浏览器插件 popup：用于管理已选容器和 LLM 配置。
- 网页内翻译弹窗：用于在当前网页中展示原文和译文。

网页内翻译弹窗支持最小化、通过悬浮入口恢复、拖拽、缩放和隐藏。关闭或最小化弹窗时，不销毁已经渲染好的翻译内容。只有用户再次在 popup 中点击“开始翻译”时，才会开启新一轮翻译并替换旧内容。

## 2. 技术栈

- 扩展框架：WXT
- UI 框架：React
- UI 组件库：shadcn/ui
- 样式方案：Tailwind CSS
- 图标库：lucide-react
- 语言：TypeScript
- 扩展模型：由 WXT 生成 Manifest V3 配置
- 本地存储：`chrome.storage.local`
- 本地存储用途：仅保存 LLM 配置和基础偏好
- 翻译缓存：MVP 暂不实现

选择 shadcn/ui、Tailwind CSS 和 lucide-react 的原因：

- popup 和网页内弹窗都需要较强的样式控制能力。
- shadcn/ui 组件轻量，适合浏览器插件的小尺寸 UI。
- Tailwind CSS 便于在 popup 和 Shadow DOM 弹窗中保持一致的设计语言。
- lucide-react 可覆盖设置、关闭、最小化、删除、刷新、测试连接等常见图标需求。
- 相比 Ant Design 或 MUI，该组合更适合 MVP 的体积控制和后续定制。

## 3. Popup 交互

### 3.1 默认页面：容器管理

popup 默认打开容器管理页面。

容器管理页面包含：

- 顶部标题。
- 右上角齿轮 icon，用于进入配置页面。
- 当前页面可用状态。
- 已选容器列表。
- 操作按钮：
  - 选择容器
  - 退出选择
  - 删除单个已选容器
  - 清空所有已选容器
  - 开始翻译

每个已选容器条目展示：

- 选择序号。
- 文本摘要。
- 状态：有效、已丢失、翻译中、已翻译、失败。
- 删除按钮。

当 LLM 配置不完整时，“开始翻译”按钮应禁用，或点击后提示用户先进入配置页补全配置。

### 3.2 配置页面

点击右上角齿轮 icon 后进入配置页面。

配置页面包含：

- Base URL。
- API Token。
- 模型名称。
- 目标语言。
- 测试连通性按钮。
- 保存按钮。
- 返回按钮。

API Token 只能保存到扩展本地存储中，不能注入网页 DOM，也不能暴露给网页脚本。

测试连通性按钮应使用当前配置发起一个小请求，并展示以下结果：

- 连接成功。
- Token 无效或未授权。
- 网络请求失败。
- 请求超时。
- 返回格式异常。

## 4. 容器选择交互

### 4.1 选择模式

当用户在 popup 中点击“选择容器”：

- content script 进入选择模式。
- 鼠标在网页上移动时，显示类似浏览器 inspect 的元素高亮框。
- 高亮框通过 overlay 实现，不直接修改目标元素的 inline style。
- 页面滚动或窗口尺寸变化后，高亮位置仍应保持准确。

### 4.2 选择规则

已选容器集合不允许存在父子嵌套关系。

具体规则：

- 点击未选择的容器：加入选择列表。
- 点击已经选择的容器：取消该选择。
- 点击已经选择的容器内部元素：取消已选择的父容器。
- 点击某个已选子容器的父容器：用父容器替换子容器。
- 清空选择后，应移除页面上的所有选择标记。

popup 和 content script 的状态必须保持同步。每次打开 popup 时，应主动向当前 active tab 查询最新选择状态。

## 5. 翻译流程

当用户点击“开始翻译”：

1. popup 校验 LLM 配置是否完整。
2. popup 通知 content script 开始新一轮翻译。
3. content script 校验每个已选元素是否仍存在于当前 document 中。
4. 已丢失的元素标记为已丢失，并在本轮翻译中跳过。
5. 如果所有已选元素都已丢失，则提示用户重新选择，不打开空翻译弹窗。
6. 对有效已选元素生成 DOM 快照。
7. 快照只保留布局、段落结构和文字相关样式。
8. 抽取文本节点，并为本轮翻译生成稳定 id。
9. 将文本节点发送给 LLM 翻译。
10. LLM 必须按文本节点 id 返回译文。
11. 将译文写入原 DOM 结构的译文克隆节点中。
12. 打开或替换网页内翻译弹窗内容。

每次点击“开始翻译”都代表新一轮翻译。MVP 不复用旧翻译结果，也不读取持久化翻译缓存。

## 6. 网页内翻译弹窗

翻译弹窗由 content script 注入当前网页。

推荐实现方式：

- 在页面中创建一个宿主节点。
- 在宿主节点上挂载 Shadow DOM。
- 在 Shadow DOM 内渲染 React 根节点。

弹窗行为：

- 默认左右两栏布局。
- 左栏展示原文克隆内容。
- 右栏展示译文克隆内容。
- 支持拖拽。
- 支持缩放。
- 支持关闭。
- 支持最小化。
- 最小化后，在页面上显示一个悬浮入口按钮。
- 点击悬浮入口按钮后，恢复弹窗。
- 关闭或最小化只隐藏弹窗，不销毁已经渲染好的内容。
- 用户再次在 popup 中点击“开始翻译”时，清空旧内容并渲染新一轮结果。

页面刷新后，所有内存状态都会丢失。用户需要重新通过 popup 开始翻译。

## 7. DOM 快照和样式范围

MVP 目标是保留阅读相关的结构和样式，而不是做到像素级复刻原网页。

需要保留：

- 布局结构。
- 段落。
- 标题。
- 列表。
- 表格。
- 代码块。
- 字体。
- 字号。
- 字重。
- 行高。
- 文字颜色。
- 间距。
- 边框。
- 基础 display 布局。

不需要保留：

- 背景图片或复杂背景。
- 动画。
- 伪元素。
- Canvas 或 WebGL 内容。
- 跨域 iframe 内容。
- 完整 Shadow DOM 重建。
- 原页面 JavaScript 行为。

MVP 中代码块默认不翻译，保持原文。

## 8. 安全约束

- 只发送用户选中容器中的文本给 LLM。
- API Token 不允许写入网页 DOM。
- API Token 不允许暴露给网页脚本。
- 克隆 DOM 时必须移除危险内容：
  - `script` 元素。
  - `onclick` 等 inline 事件属性。
  - `javascript:` 等危险 URL。
- 不允许让 LLM 生成 HTML。
- LLM 只接收结构化文本节点数据，并返回结构化文本节点译文。

## 9. MVP 暂不包含

- 翻译缓存。
- 跨页面翻译恢复。
- 页面刷新后基于 selector 重新定位元素。
- 跨域 iframe 支持。
- 完整 Shadow DOM 支持。
- 像素级样式复刻。
- 背景保留。
- 离线翻译。
- Side Panel。

## 10. 异常处理

MVP 需要明确处理以下情况：

- 当前页面无法注入脚本。
- LLM 配置缺失。
- API Token 无效。
- LLM 请求超时。
- LLM 返回非法 JSON。
- LLM 返回结果缺少某些文本节点 id。
- 已选容器在翻译前丢失。
- 所有已选容器都已丢失。
- 已选容器没有可翻译文本。
- 已选容器内容过大，超出 MVP 限制。

建议 MVP 限制：

- 单次最多选择容器数：10 个。
- 单次最多文本节点数：500 个。
- 单次最多字符数：30,000 个。
- 请求超时时间：60 秒。

如果用户选择的内容超过限制，应提示用户选择更小的容器。
